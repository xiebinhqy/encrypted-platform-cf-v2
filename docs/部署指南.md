## 项目简介

Encrypted Notes v2 是一款**端对端加密私人笔记**应用，所有笔记内容在用户的浏览器中使用 AES-GCM 加密后再上传到服务器，服务器无法读取任何笔记内容，即使是服务提供者也看不到用户的笔记。

> **核心安全理念**：数据在传输前加密，服务器仅作为加密数据的存储容器，解密密钥仅存在于用户浏览器中。

## 前置要求

### 1. 注册 Cloudflare 账号

访问 [https://dash.cloudflare.com/](https://dash.cloudflare.com/) 注册账号。

### 2. 安装 Node.js

```bash
# 检查版本（需要 >= 18.x）
node --version
npm --version
```

### 3. 安装 Wrangler CLI

```bash
npm install -g wrangler
```

### 4. 登录 Cloudflare

```bash
wrangler login
```

浏览器会自动打开登录页面，授权后终端显示 `Successfully logged in`。

### 5. 创建 D1 数据库

```bash
# 创建生产库
wrangler d1 create notes-db

# 创建测试库
wrangler d1 create staging-notes-db
```

创建成功后，将输出的 `database_id` 更新到 `backend/wrangler.toml` 中对应的 `[[d1_databases]]` 配置。

## 快速部署（Cloudflare Workers）

### 第一步：克隆项目

```bash
git clone https://github.com/你的用户名/encrypted-notes-v2.git
cd encrypted-notes-v2
```

### 第二步：配置后端

编辑 `backend/wrangler.toml`，主要配置项：

```toml
# ========== 基础配置 ==========
name = "encrypted-notes-backend"       # Worker 名称
main = "src/index.js"                  # 入口文件

# ========== 静态资源服务（经典版前端） ==========
[assets]
directory = "../frontend/classic"      # 前端目录
binding = "ASSETS"

# ========== 全局变量 ==========
[vars]
ENVIRONMENT = "production"             # environment/staging/development
ALLOW_PUBLIC_REGISTRATION = "true"     # 是否允许公开注册
PBKDF2_ITERATIONS = "100000"           # 加密迭代次数
ADMIN_EMAIL = "your@email.com"         # 管理员邮箱
FRONTEND_DOMAIN = "https://notes.yourdomain.com"  # 前端域名

# ========== D1 数据库绑定 ==========
[[d1_databases]]
binding = "DB"
database_name = "notes-db"
database_id = "你的数据库ID"            # 替换为实际ID
```

### 第三步：部署到生产环境

```bash
cd backend

# 部署到生产环境
npx wrangler deploy --env production

# 部署到测试环境
npx wrangler deploy --env staging
```

### 第四步：初始化数据库

```bash
# 执行数据库迁移
npx wrangler d1 execute notes-db --file=./migrations/0000_init.sql --env production

# 测试环境
npx wrangler d1 execute staging-notes-db --file=./migrations/0000_init.sql --env staging
```

### 第五步：配置自定义域名

在 Cloudflare Dashboard → Workers 路由中添加：

```
notes.yourdomain.com/* → encrypted-notes-backend.production
```

然后在 DNS 设置中添加 CNAME 记录指向 Worker。

## 本地开发环境搭建

### 1. 安装依赖

```bash
cd backend
npm install
```

### 2. 创建本地 D1 数据库

```bash
cd backend
npx wrangler d1 create local-notes-db

# 将输出的 database_id 更新到 wrangler.toml 的 [dev] 或默认 d1_databases 配置中
```

> 注意：本地开发使用 `wrangler dev` 时，D1 数据库会自动在本地 `.wrangler` 目录中创建 SQLite 文件，无需额外配置。

### 3. 初始化本地数据库

```bash
cd backend
npx wrangler d1 execute DB --file=./migrations/0000_init.sql --local
```

### 4. 启动开发服务器

```bash
cd backend
npx wrangler dev --port 8787
```

### 5. 访问应用

打开浏览器访问 `http://localhost:8787` 即可看到经典版前端界面。

> **自动检测**：前端会自动判断是否在本地环境，如果是则 API 地址为 `http://localhost:8787`，否则为线上地址。

## 数据库初始化与迁移

### 数据库表结构

```sql
-- 用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    public_key TEXT NOT NULL UNIQUE,         -- 用户公钥（密钥哈希）
    recovery_code_hash TEXT NOT NULL,        -- 恢复码哈希
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 笔记表
CREATE TABLE notes (
    id TEXT PRIMARY KEY,                     -- UUID
    user_id INTEGER NOT NULL,
    title TEXT NOT NULL,                     -- 加密后的标题
    content TEXT NOT NULL,                   -- 加密后的内容
    category TEXT DEFAULT '',                -- 加密后的分类
    tags TEXT DEFAULT '',                    -- 加密后的标签
    revision_count INTEGER DEFAULT 1,        -- 修改次数
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 分类表
CREATE TABLE categories (
    id TEXT PRIMARY KEY,                     -- UUID
    user_id INTEGER NOT NULL,
    name TEXT NOT NULL,                      -- 加密后的分类名
    color TEXT DEFAULT '',
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 分享表
CREATE TABLE shares (
    id TEXT PRIMARY KEY,                     -- 分享码
    note_id TEXT NOT NULL,
    user_id INTEGER NOT NULL,
    max_views INTEGER DEFAULT 0,             -- 最大查看次数
    view_count INTEGER DEFAULT 0,            -- 当前查看次数
    expires_at DATETIME,                     -- 过期时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (note_id) REFERENCES notes(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 数据库迁移

如果已有数据库需要增加新列（初次部署无需操作，初始化脚本已包含完整表结构）：

```sql
-- 添加 tags 列（v2.1.0 新增）
ALTER TABLE notes ADD COLUMN tags TEXT DEFAULT '';

-- 添加 revision_count 列（v2.1.0 新增）
ALTER TABLE notes ADD COLUMN revision_count INTEGER DEFAULT 1;
```

> **兼容性说明**：后端代码已内置自动迁移功能，如果查询时发现列不存在会自动添加，无需手动执行上述 SQL。

## 域名配置与 HTTPS

### 自定义域名绑定

1. 在 Cloudflare Dashboard → Workers & Pages → 你的 Worker → Triggers
2. 添加自定义域名路由，例如 `notes.yourdomain.com/*`
3. 在 DNS 设置中添加 CNAME 记录

### 前端域名配置

修改 `frontend/shared/api/index.js` 中的默认 API 地址：

```javascript
const getApiBase = () => {
  const host = window.location.hostname;
  if (host === "localhost" || host === "127.0.0.1") {
    return `http://${host}:8787`;   // 本地开发
  }
  return "https://api.yourdomain.com";  // 线上环境
};
```

### HTTPS

Cloudflare Workers 默认提供 HTTPS，无需额外配置。但需要注意：

- Worker 部署后自动获得 `*.workers.dev` 域名的 HTTPS
- 自定义域名需要确保 Cloudflare 代理（橙色云朵）已开启

### 安全注意事项

1. **加密模块不可修改**：`frontend/shared/crypto/` 目录下的任何文件都**绝对不能修改**，否则会导致数据无法解密
2. **主密钥管理**：用户的主密钥仅存在于浏览器的内存中，服务器永远不会存储或传输主密钥
3. **恢复码**：每个用户注册时会生成一次性恢复码，用于重置主密钥，务必提醒用户保存
4. **HTTPS 强制**：生产环境必须使用 HTTPS，防止中间人攻击
5. **CORS 配置**：如果前端和后端分开部署，需要正确配置 CORS 头

### 性能建议

| 场景 | 建议 |
|------|------|
| **100+ 笔记** | 无需额外优化，当前架构可流畅运行 |
| **500+ 笔记** | 启用后端分页查询，限制单次返回笔记数量 |
| **1000+ 笔记** | 建议启用虚拟滚动、IndexedDB 本地缓存、仪表盘懒加载 |
| **高并发场景** | 增加 D1 数据库的读取缓存、使用 KV 存储热数据 |

## 常见问题排查

### Q: 部署后访问显示 404

**原因**：静态资源路径配置不正确

**解决**：
```bash
# 确认 frontend/classic/ 目录中存在 index.html
ls frontend/classic/

# 重新部署
cd backend && npx wrangler deploy
```

### Q: GET /notes 返回 500

**原因**：数据库表中缺少 `tags` 或 `revision_count` 列

**解决**：后端已内置自动迁移逻辑，通常重启即可。如果问题持续，手动执行：
```bash
npx wrangler d1 execute notes-db --command="ALTER TABLE notes ADD COLUMN tags TEXT DEFAULT ''" --env production
npx wrangler d1 execute notes-db --command="ALTER TABLE notes ADD COLUMN revision_count INTEGER DEFAULT 1" --env production
```

### Q: 用户能登录但笔记列表空白

**原因**：前端 API 地址与后端不匹配

**解决**：检查 `frontend/shared/api/index.js` 中的 `API_BASE` 是否正确指向后端地址。

### Q: 创建分类后导航栏不显示

**原因**：后端返回的字段名与前端期望的不一致（如 `name` vs `name_cipher`）

**解决**：确认后端的 `compatGetCategories` 返回的是 `{ id, name_cipher }` 格式。如果使用的是 v2 原生路由，请确保字段名映射正确。

### Q: 如何备份数据？

**方法一（推荐）**：使用应用的「导出」功能，选择「密文导出」，下载 JSON 文件保存
**方法二**：使用 `wrangler d1 backup` 备份整个数据库

## 版本升级指南

### 从 v1 升级到 v2

1. 部署 v2 后端代码到新的 Worker
2. 在 v2 的 `wrangler.toml` 中绑定 v1 的 D1 数据库
3. 运行数据库迁移脚本添加新字段
4. 验证兼容路由正常工作
5. 更新 DNS 记录指向新 Worker

### 小版本升级（v2.x.x）

```bash
git pull origin main
cd backend
npx wrangler deploy --env production
npx wrangler d1 execute notes-db --file=./migrations/XXXX_migration.sql --env production
```
