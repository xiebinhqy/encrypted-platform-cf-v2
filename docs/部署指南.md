## 项目简介

Encrypted Notes v2 是一款**端对端加密私人笔记**应用，所有笔记内容在用户的浏览器中使用 AES-GCM 加密后再上传到服务器，服务器无法读取任何笔记内容，即使是服务提供者也看不到用户的笔记。

> **核心安全理念**：数据在传输前加密，服务器仅作为加密数据的存储容器，解密密钥仅存在于用户浏览器中。

## 前置要求

### 1. 注册 Cloudflare 账号

访问 [https://dash.cloudflare.com/](https://dash.cloudflare.com/) 注册账号。

### 2. 安装 Node.js

```bash
# 检查版本（需要 >= 18.x）
node --version
npm --version
```

### 3. 安装 Wrangler CLI（Cloudflare 部署用）

```bash
npm install -g wrangler
```

### 4. 安装 Docker（本地部署用）

访问 [https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/) 下载 Docker Desktop。

### 5. 登录 Cloudflare

```bash
wrangler login
```

浏览器会自动打开登录页面，授权后终端显示 `Successfully logged in`。

### 6. 创建 D1 数据库（Cloudflare 部署用）

```bash
# 创建生产库
wrangler d1 create notes-db

# 创建测试库
wrangler d1 create staging-notes-db
```

创建成功后，将输出的 `database_id` 更新到 `backend/wrangler.toml` 中对应的 `[[d1_databases]]` 配置。

## 方案一：Cloudflare Workers 部署（线上）

### 第一步：克隆项目

```bash
git clone https://github.com/你的用户名/encrypted-notes-v2.git
cd encrypted-notes-v2
```

### 第二步：配置后端

编辑 `backend/wrangler.toml`，主要配置项：

```toml
name = "encrypted-notes-backend"
main = "src/index.js"

[assets]
directory = "../frontend/classic"
binding = "ASSETS"

[vars]
ENVIRONMENT = "production"
ALLOW_PUBLIC_REGISTRATION = "true"
PBKDF2_ITERATIONS = "100000"
ADMIN_EMAIL = "your@email.com"
FRONTEND_DOMAIN = "https://notes.yourdomain.com"

[[d1_databases]]
binding = "DB"
database_name = "notes-db"
database_id = "你的数据库ID"
```

### 第三步：部署到生产环境

```bash
cd backend
npx wrangler deploy --env production  # 生产环境
npx wrangler deploy --env staging     # 测试环境
```

### 第四步：初始化数据库

```bash
npx wrangler d1 execute notes-db --file=./migrations/0000_init.sql --env production
```

### 第五步：配置自定义域名

在 Cloudflare Dashboard → Workers & Pages → 你的 Worker → Triggers 中配置域名路由。

## 方案二：Docker 本地部署

### 快速启动（Windows）

```batch
start-docker.bat
```

### 手动启动

```bash
# 1. 修改 .env 中的 JWT_SECRET
# 2. 构建并启动
docker compose up -d

# 3. 访问 http://localhost:3000
```

### 查看日志

```bash
docker compose logs -f
```

### 停止服务

```bash
docker compose down
# 如需同时清除数据卷：
docker compose down -v
```

### 镜像拉取

```bash
docker pull ghcr.io/xiebinhqy/encrypted-platform-cf-v2:latest

docker run -d -p 3000:3000 \
  -e JWT_SECRET=你的密钥 \
  -v notes-data:/app/backend/data \
  ghcr.io/xiebinhqy/encrypted-platform-cf-v2:latest
```

### 从 Cloudflare D1 迁移到本地 SQLite

```bash
# 1. 从 D1 导出数据
npx wrangler d1 execute notes-db --command="SELECT * FROM notes" --env production --json > notes_export.json

# 2. 导入到本地 SQLite
node scripts/migrate-from-d1.js ./data/notes.db ./notes_export.json
```

## 数据库初始化与迁移

### 数据库表结构

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    public_key TEXT NOT NULL UNIQUE,
    recovery_code_hash TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE notes (
    id TEXT PRIMARY KEY,
    user_id INTEGER NOT NULL,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    category TEXT DEFAULT '',
    tags TEXT DEFAULT '',
    revision_count INTEGER DEFAULT 1,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE categories (
    id TEXT PRIMARY KEY,
    user_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    color TEXT DEFAULT '',
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 自动迁移

后端内置自动迁移功能（`database.js` v14），首次启动时自动建表并执行必要的列迁移。

## 域名配置与 HTTPS

### Cloudflare Workers
- Workers 自动获得 `*.workers.dev` 域名的 HTTPS
- 自定义域名需确保 Cloudflare 代理（橙色云朵）已开启

### Docker 部署
- 本地部署（localhost）：HTTP 可接受
- 远程部署：必须配置 HTTPS + Nginx 反向代理

### 部署后

- [ ] 注册/登录正常
- [ ] 创建笔记/分类正常
- [ ] 三层密码体系工作正常
- [ ] 备份/恢复/导入导出正常
- [ ] 亮色/暗色主题切换正常

### 安全注意事项

1. **加密模块不可修改**：`frontend/shared/crypto/` 目录绝对不能修改
2. **JWT 密钥**：Docker 部署必须修改 `.env` 中的 `JWT_SECRET`
3. **恢复码**：每个用户注册时会生成一次性恢复码，务必提醒用户保存
4. **数据库备份**：定期备份 SQLite 文件（Docker volume）

## 常见问题排查

### Q: Docker 构建失败
```bash
docker compose build --no-cache
```

### Q: 创建笔记返回 500
后端内置自动迁移逻辑，重启即可。如果持续，检查数据库是否存在旧列名约束。

### Q: 端口被占用
```bash
netstat -ano | findstr :3000
set PORT=3001 && docker compose up -d -e PORT=3001
```

### Q: 如何备份数据？
- 方法一：使用应用的「导出」功能
- 方法二：Docker volume 数据备份
