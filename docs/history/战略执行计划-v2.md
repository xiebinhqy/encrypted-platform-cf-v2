## 📐 执行计划

### 阶段一：止血（优先完成）

#### 任务 1.1：修复"保存后笔记不显示"（3.6）
**问题：** 创建/编辑笔记后，笔记列表不更新，或标题显示为"未命名笔记"
**根因分析：**
- 后端返回 `title_cipher` 字段，但前端可能仍在读取 `title` 字段
- 创建笔记后未正确刷新列表缓存

**执行步骤：**
1. 检查 `note.api.js` 中 `createNote` / `updateNote` 的返回值处理
2. 检查前端 `app.js` 中保存后的回调逻辑
3. 确保保存成功后调用 `rebuildNotesListCache`
4. 测试：创建笔记 → 验证列表立即更新

**验证标准：**
- [ ] 创建笔记后，列表立即显示新笔记
- [ ] 编辑笔记标题后，列表标题同步更新
- [ ] 标题不是"未命名笔记"

### 阶段三：扩展（IndexedDB 离线支持）

#### 任务 3.1：前端 IndexedDB 离线缓存
**目标：** 断网时仍可查看/编辑笔记，网络恢复后自动同步

**架构设计：**
```
用户操作 → IndexedDB（本地缓存）
       ↓ 网络可用时
后端 KV ←→ D1
```

**执行步骤：**
1. 创建 `frontend/shared/offline/db.js` — IndexedDB 封装
2. 创建 `frontend/shared/offline/sync.js` — 同步队列管理
3. 修改 `note.api.js` — 写操作先进 IndexedDB，后台同步到服务器
4. 实现冲突解决策略（Last-Write-Wins 或 用户选择）

**验证标准：**
- [ ] 断网时创建笔记 → 网络恢复后自动上传
- [ ] 断网时编辑笔记 → 网络恢复后自动同步
- [ ] 多端编辑冲突时有提示

---

#### 任务 3.2：降温 Cron Job
**目标：** 超过 7 天未访问的笔记，从 KV 降温到 D1

**实现方式：** Cloudflare Cron Trigger

**执行步骤：**
1. 在 `wrangler.toml` 添加 Cron 配置
2. 创建 `backend/src/cron/cooldown.js` — 降温逻辑
3. 扫描 KV 中超过 7 天未访问的笔记，删除 KV 缓存
4. 确保下次访问时自动从 D1 回源到 KV

**验证标准：**
- [ ] Cron Job 每天执行一次
- [ ] 超过 7 天的笔记从 KV 删除
- [ ] 访问降温笔记时自动从 D1 加载并回写 KV
