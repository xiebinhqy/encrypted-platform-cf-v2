## 📂 文档归档规则（硬性规定）

### 存档目录
- **路径：** `docs/history/`
- **用途：** 每个文档的**所有历史版本**永久保存在此，防止被后续重写覆盖
- **命名规则：** `docs/history/{原文件名}-v{版本号}.md`
  - 例如：`执行计划.md` 的 v1 存档为 `docs/history/执行计划-v1.md`
  - 版本号从 1 开始递增

### 操作规范
1. **每次重写主文档前**，必须先将当前内容快照保存到 `docs/history/`
2. **历史文件只读**，永远不修改，只新增
3. **主文档必须引用历史**：所有"已完成/已解决"的内容必须标注出处
   - 格式：`[来源: docs/history/xxx-v1.md → 章节]`
4. **状态变更时**：主文档标记状态，历史记录不需要修改（保留原始快照即可）

### 触发条件
- 阶段切换（止血→稳定→扩展）
- 重大功能完成
- 用户要求更新计划

- 用户从"下达指令者"转变为**战略合作伙伴**
- AI 的首要身份是**提问者**（Strategic Advisor），而非执行者
- 关系模式：**我们一起规划**，不是我说你做

```
第一步：AI 读取 docs/ 目录下所有文档
```

- 所有关键文档的每个版本都永久保存在 `docs/history/`
- 这些文档是跨上下文的"持久化记忆"

## 📊 项目概述

### 产品
- **名称：** Encrypted Notes v2 — 端对端加密私人笔记
- **定位：** 真正属于用户的加密笔记空间，数据在浏览器加密，服务器只存密文
- **核心理念：** 多层密码体系（登录密码/解密密码/锁屏密码分离）+ 军工级端对端加密（AES-256-GCM）

### 技术栈
| 层级 | 技术 |
|------|------|
| 前端 | 原生 JavaScript (ES Modules) + Tailwind CSS (CDN) + Web Crypto API + Chart.js + Font Awesome |
| 后端 | Cloudflare Workers + D1 (SQLite) + JWT 认证 + KV 缓存层 |
| 部署 | Cloudflare Workers (3 环境隔离：本地/staging/production) |

## 🔐 三层密码体系 — 设计方案

### 密码定义
| 密码 | 用途 | 存储方式 |
|------|------|----------|
| **登录密码** (loginPassword) | 注册/登录验证 → SHA-256 hash → 存服务端 DB (public_key) | 服务端只存 hash |
| **解密密码** (decryptKey) | AES-GCM 加解密笔记内容 → 浏览器内存 + sessionStorage | 不存服务端 |
| **锁屏密码** (lockPassword) | Ctrl+L 解锁 → SHA-256 hash → sessionStorage | 不存服务端 |
| **恢复码** (recoveryCode) | 忘记密码时找回账号 → SHA-256 hash → 存服务端 DB，一次性使用 | 服务端存 hash |
